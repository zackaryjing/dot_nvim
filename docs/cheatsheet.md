# Neovim 个人速查表

## 消息

| 快捷键或命令 | 用途 |
| --- | --- |
| `<leader>snh` | 查看 Noice 消息历史 |
| `<leader>snl` | 查看最后一条 Noice 消息 |
| `<leader>sna` | 查看全部 Noice 消息 |
| `<leader>n` | 查看 Snacks 通知历史 |
| `<leader>un` | 关闭浮动通知 |
| `:messages` | 查看 Neovim 原生消息历史 |

`<leader>` 默认为空格键。

## 命令行

| 快捷键 | 用途 |
| --- | --- |
| 输入 `:` 命令时按 `↓` / `↑` | 浏览较新 / 较旧的历史命令 |
| 输入 `:` 命令时按 `<C-n>` / `<C-p>` | 选择下一个 / 上一个候选命令 |

## 终端

| 快捷键 | 用途 |
| --- | --- |
| `<leader>ft` | 打开或隐藏项目根目录终端 |
| `<leader>fT` | 打开或隐藏当前工作目录终端 |
| `<C-/>` | 快速聚焦或隐藏项目根目录终端 |
| `<C-h/j/k/l>` | 在左 / 下 / 上 / 右窗口之间移动 |

## 主题

| 快捷键 | 用途 |
| --- | --- |
| `<leader>ub` | 在 dark / light 背景之间切换 |
| `<leader>uC` | 打开 colorscheme 选择器并预览主题 |

默认使用 `tokyonight-day`。个人配置将其主编辑区背景覆盖为纯白色，配置位于
`lua/plugins/colorscheme.lua`。

## 注释

| 快捷键 | 用途 |
| --- | --- |
| `gcc` | 注释或取消注释当前行 |
| 可视模式中按 `gc` | 注释或取消注释选中区域 |
| `gcap` | 注释或取消注释当前段落 |

## 跳转

| 快捷键 | 用途 |
| --- | --- |
| `<leader>j` | 使用 Flash 在可见范围内快速跳转 |
| `s` | 删除当前字符并进入 Insert 模式，保留 Vim 原生行为 |
| `S` | 使用 Flash Treesitter 按代码结构跳转或选择 |

## LSP

| 快捷键 | 用途 |
| --- | --- |
| `gd` | 跳转到定义 |
| `gD` | 跳转到声明 |
| `gr` | 查看引用 |
| `gI` | 跳转到实现 |
| `gy` | 跳转到类型定义 |
| `K` | 显示悬浮文档 |
| `<leader>cl` | 查看当前 LSP 信息 |

## Python

| 快捷键 | 用途 |
| --- | --- |
| `<leader>cv` | 为当前项目选择 Python 虚拟环境 |

## 诊断

| 快捷键 | 用途 |
| --- | --- |
| `<leader>cd` | 在浮窗中显示当前行的完整诊断 |
| `]d` / `[d` | 跳转到下一个 / 上一个诊断 |
| `]e` / `[e` | 跳转到下一个 / 上一个错误 |
| `]w` / `[w` | 跳转到下一个 / 上一个警告 |
| `<leader>xx` | 使用 Trouble 查看整个项目的诊断列表 |
| `<leader>xX` | 使用 Trouble 查看当前文件的诊断列表 |

## Snippet

| 操作 | 用途 |
| --- | --- |
| 在 C++ 文件中输入 `ltcd` 并确认补全 | 插入剪贴板中的 Solution，并将光标停在 problem 位置 |
| 选中文本后按 `c`，输入 `wvar` 并确认补全 | 插入 `cout << "变量名: " << 变量名 << endl;` |
| 选中文本后按 `c`，输入 `wline` 并确认补全 | 插入 `cout << 表达式 << endl;` |
| 输入 `tv` 并确认补全 | 插入 `temp_vector()`，并将光标停在括号内 |
| `Tab` | 跳转到下一个 snippet 占位符 |
| `Shift-Tab` | 跳转到上一个 snippet 占位符 |

## C++

| 快捷键 | 用途 |
| --- | --- |
| `<leader>rr` | 保存、编译并在浮动终端运行当前 C++ 文件，产物写入项目的 `output/` |
| `<leader>rc` | 保存、编译并在底部终端运行当前 C++ 文件，产物写入项目的 `output/` |
| `<leader>cf` | 使用 `clang-format` 格式化当前文件 |
| `gd` | 使用 `clangd` 跳转到定义 |

运行窗口会显示编译结果、程序输出和退出码。程序运行时可以直接输入 `cin` 内容。
使用 `<C-j>` 从代码窗口进入底部运行窗口，使用 `<C-k>` 返回代码窗口。

## 剪贴板与补全

| 快捷键 | 用途 |
| --- | --- |
| `<leader>p` | 粘贴系统剪贴板内容 |
| 在可视模式中按 `<leader>y` | 将选中内容复制到系统剪贴板 |
| `Tab` | 确认当前补全候选；无补全候选时保留原有缩进功能 |
| `Shift-Tab` | 在 snippet 中反向跳转占位符 |

## 自动保存

普通文件会每 5 秒自动保存，并在窗口失焦或离开 buffer 时立即保存。
自动保存不会触发格式化；需要格式化时使用 `<leader>cf`。

## 配置重载

修改 `lua/plugins/` 中的插件规格后，退出并重新启动 Neovim 最可靠：

```vim
:qa
```
